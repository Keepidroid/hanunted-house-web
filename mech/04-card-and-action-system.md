# 卡牌与行动系统

> 拆分来源：[core-mechanics.md](../core-mechanics.md)
>
> 说明：本文档聚焦三类核心牌堆、埋牌规则、拥有权、交易限制与特殊行动使用限制。

## 1. 牌堆与状态模型

核心运行中牌堆包括：

- `Event Deck`
- `Item Deck`
- `Omen Deck`

`Scenario Deck` 只用于开局选定本局作祟映射，不作为局中常规抽牌牌堆。

牌的核心状态建议至少支持：

- 所属牌堆
- 当前持有者
- 是否处于牌堆底部

第三版的重要特点：

- Event 牌结算后不是丢弃，而是 `bury` 到 Event 牌堆底部

## 2. Event

处理规则：

- 抽到后朗读斜体 flavor 文本并结算规则
- 若牌要求 trait roll，则按结果表读对应结果
- 结算结束后，将该牌 `bury` 到 Event 牌堆底部

实现重点：

- Event 是一次性效果，不形成持有关系
- `bury` 应视为统一牌堆操作，而不是特殊事件例外

## 3. Item

处理规则：

- 抽到后朗读文本
- 正面朝上放在玩家面前
- 表示该探索者当前持有该物品

实现重点：

- Item 是显式公开持有
- 后续可以被交易、使用、搜刮尸体转移

## 4. Omen

处理规则：

- 抽到后朗读文本
- 正面朝上放在玩家面前
- 每次抽到预兆牌后，立刻进行一次 `haunt roll`

作祟触发相关约束：

- `haunt roll` 使用“全场当前持有的预兆牌总数”作为骰子数量
- 结果 `5+` 时作祟开始
- 执行该次 `haunt roll` 的玩家称为 `haunt revealer`
- 如果抽到了最后一张预兆牌且作祟尚未开始，则自动开始作祟

完整作祟流程见 [05-haunt-engine.md](./05-haunt-engine.md)。

## 5. 交易规则

标准交易约束：

- 每回合至多交易一次
- 必须与同一空间内的另一名探索者交易
- 需要双方同意
- 可以交换任意数量的 Item 与 Omen
- 不要求等价交换

限制条件：

- 如果某件 Item / Omen 本回合已经被你用于其特殊行动，则本回合不能再把它给别人
- 如果某件牌是武器且你已用它攻击过，本回合不能交易它

## 6. 特殊行动规则

许多 Item / Omen 带有 special action 图标。

通用规则：

- 特殊行动永远是可选的
- 除非明确说明，否则每个特殊行动每回合最多使用一次
- 只能使用你在自己回合开始时就已持有的 Item / Omen 上的特殊行动
- 没有图标、但写着“只要你持有就生效”的能力，属于常驻效果，不受上述限制

这意味着系统需要同时区分：

- “带图标的主动特殊行动”
- “只要持有就生效的被动常驻效果”
